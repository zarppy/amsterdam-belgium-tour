# Deployment Guide: Hosting on GitHub Pages

Follow these steps to host your **Netherlands & Belgium Tour (2026 Edition)** website on GitHub Pages. The project is already structured for root deployment.

## 1. Project Status Check
Your project files are located in the root of the `tour` repository. 
- **Main Entry**: `index.html`
- **Styles**: `styles.css`
- **Logic & Map**: `script.js`
- **Assets**: `images/` (Contains the new `hero-bg.png` and local attraction shots)

---

## 2. Commit and Push to GitHub
Stage and push your changes to your `origin` repository. Ensure you are in the `tour` directory.

```powershell
# Stage all changes
git add .

# Commit with the final itinerary update
git commit -m "Finalize 10-day Netherlands-Belgium loop with restaurant bucket list"

# Push to GitHub
git push origin main
```

---

## 3. Enable GitHub Pages
1. Go to your repository on GitHub: [github.com/zarppy/amsterdam-copenhagen-tour](https://github.com/zarppy/amsterdam-copenhagen-tour)
2. Click on **Settings** (top tab).
3. Click on **Pages** in the left-hand sidebar.
4. Under **Build and deployment > Branch**, ensure `main` is selected and the folder is `/ (root)`.
5. Click **Save**.

---
