# Deployment Guide: Hosting on GitHub Pages

Follow these steps to host your **Netherlands & Belgium Tour** website on your personal GitHub site (`zarppy`).

## 1. Prepare Your Repository Structure
Currently, your project files are inside an `amsterdam-copenhagen` folder within the `tour` repository. For the cleanest URL (e.g., `https://zarppy.github.io/amsterdam-belgium-tour/`), it is recommended to move the files to the **root** of the repository.

### Option A: Move to Root (Recommended)
Run these commands in your terminal from the `tour` directory:
```powershell
# Move files from subfolder to root
Move-Item -Path .\amsterdam-copenhagen\* -Destination .\ -Force
# Remove the now empty subfolder
Remove-Item -Path .\amsterdam-copenhagen\ -Recurse
```

### Option B: Keep Subfolder
If you keep the files in the subfolder, your URL will be:
`https://zarppy.github.io/amsterdam-copenhagen-tour/amsterdam-copenhagen/`

---

## 2. Commit and Push to GitHub
Once your structure is decided, commit the changes and push them to your `origin` repository.

```powershell
# Stage all changes (including deletions and new files)
git add -A

# Commit with a descriptive message
git commit -m "Finalize Netherlands & Belgium Tour itinerary and structure"

# Push to GitHub
git push origin main
```

---

## 3. Enable GitHub Pages
1. Go to your repository on GitHub: [github.com/zarppy/amsterdam-copenhagen-tour](https://github.com/zarppy/amsterdam-copenhagen-tour)
2. Click on **Settings** (top tab).
3. Click on **Pages** in the left-hand sidebar.
4. Under **Build and deployment > Branch**, ensure `main` is selected and the folder is `/ (root)`.
5. Click **Save**.

---

## 4. Verify Your Site
After a minute or two, your site will be live at:
**`https://zarppy.github.io/amsterdam-copenhagen-tour/`**
*(Or with `/amsterdam-copenhagen/` appended if you chose Option B)*

### Checklist for Success:
- [ ] **Images**: Ensure all images load correctly. If some images are missing, check that the paths in `index.html` match the files in the `images/` folder.
- [ ] **Map**: Verify the Leaflet map renders and display pins for both Netherlands and Belgium.
- [ ] **Navigation**: Test the top navigation links to ensure they jump to the correct days.
